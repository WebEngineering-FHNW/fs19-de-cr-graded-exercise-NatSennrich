Web Engineering Projekt:
Restaurant El Asador - Orders Manager

Diese Seite gilt als Beschreibung meines Projektes

Einführung

Restaurant El Asador - Orders Manager ist eine Grails basierte Webapplikation inspiriert vom Restaurant meiner Mutter und hat das Ziel den Bestellungprozess im Betrieb zu vereinfachen, denn aktuell werden die Bestellungen nur auf Papier geschrieben.
Was diese Webapplikation anbietet is:

  • Übersicht von verfügbaren Tischen im Restaurant
  • Eine Liste von offenen Bestellung
  • Ein Bereich für die Erfassung neue Bestellungen
  • Übersicht von allen Bestellungen für ein bestimmtes Datum
  • Eine Bestanaufnahme-Seite der Gerichten, sowie der Getränke
  • SpringSecurity Funktionalität

Anleitung

Anleitung für die wichtigsten Funktionalitäten

Sich einlogen
Um sich einzulogen verwenden klicken Sie auf den Knop "Zur Applikation hier klicken" am Ende dieser Seiten. Dann geben Sie eine dieser Daten an:
Username Password
user user
admin admin
Neue Bestellung erstellen
Um eine neue Bestellung zu erstellen müssen Sie
  1. Auf einen freien Tisch klicken (nicht rot)
  2. Ein Produkt hinzufügen: klick auf dem Essen oder Getränk Icon
  3. Produkt aus der Liste auswählen
  4. Anzahl angeben
  5. Auf Knopf "Save" klicken
  6. Die Bestellung kann jetzt verlassen, bezahlt oder sogar gelöscht werden (Klick auf dem gewünschten Knopf
Offene Bestellung bearbeiten
Auf der Index-Seite klick auf die gewünschte Bestellung aus der rechten Liste oder direkt auf dem Tisch mit der Bestellung. Dann Änderungen vornehmen.

Ein neues Produkt zur Bestandaufnahme hinzufügen
  1. In der Navigation, auf BeverageInventory oder FoodInventory klicken
  2. Daten vom neuen Produkt ins Formular eintragen
  3. Auf save klicken
Sich abmelden
Um sich aus der Applikation abzumelden, einfach auf dem "Logout" Knopf klicken.

Verwendete Layouts

Um alle Seite der Applikation einheitlich zu gestalten wurden zwei Layouts verwendet: Header und Table Overview

Header Layout
Das Header Layout wird in allen Seiten verwendet und beinhaltet die basis HTML Struktur für alle Seite, sowie die Importierung von CSS Stylesheets und JavaScripts. Das Logo des Kunden (Restaurant El Asador in diesem Fall) und die Navigation sind in diesem Layout vorhanden.
TableOverview Layout
Dieses Layout wird in den Views orderManager/index und orderRestaurant/newOrder und es ist verantwortlich für die Darstellung der verfügbaren Tischen im Restaurant. Die dargestellten Tischen sind klickbar und lösen die Aktion "neue Bestellung erstellen" falls der Tisch frei ist oder "Bestellung bearbeiten" falls dieser Tisch eine offene Bestellung hat. Dieses Layout verwendet zudem das Header Layout.
index
Die erste Seite nach dem Login. Diese Seite verwenden die Layouts Header und TableOverview. Auf dieser Seite bekommt man einen Übersicht von den offenen Bestellungen.
newOrder
Die Seite auf der man eine Neue Bestellung und OrderItems erstellen kann, oder eine offene Bestellung bezahlen oder löschen.
orderHistory
Diese Seite bietet eien Übersicht von allen Bestellungen mit den drei verschienen Status sowie einen DatePicker, damit man ein anderes datum auswählen kann.
beverageInventory und foodInventory
Auflistung von allen verfügbaren Getränk- bzw. Essenprodukte. Sowie ein Formular für die Erstellung neue Produkte.

Domain Classes

Damit die Webapplikation richtig funktioniert wurden folgenden Domain Classes implementiert:

BeverageProduct

Diese Klasse dient für die Instanzierung eindeutiger Getränke. Jede Instanz diese Klasse hat folgende Eigenschaften: Name, Kategorie, Behälter, Preis und das Datum, wann sie zuletzt gespeichert wurde.

FoodProduct

Diese Klasse dient für die Instanzierung eindeutiger Gerichte. Jede Instanz diese Klasse hat folgende Eigenschaften: Name, Beschreibung, Kategorie, Preis und das Datum, wann sie zuletzt gespeichert wurde.

OrderItem

Für die Bestellungen werden Instanzen dieser Klasse erzeugt. Man kann ein Getränk oder ein Gerich einem OrderItem zuordnen und dieser wird mit Anzahl und Betrag gespeichert.

RestaurantOrder

Jede Bestellung ist eine Instanz dieser Klasse. Eine Bestellung besteht aus einen oder mehreren OderItem, eine Tischnummer, Bestellungsnummer, User, Start- und Enddatum und ein Status (Offen, Bezahlt oder Storniert).

User, Role und UserRole

Diese Klasse sind notwendig für die User Verwaltung. Mehr Info in Sektion User Verwaltung.

Controllers

Beschreibung von angewendeten Controllers

BeverageProductController

Zuständig für die Verwaltung von Getränken. Die Methoden dieses Controllers erlauben:

  • Erstellung eine Liste von verfügbaren Getränke
  • Erstellung neue Getränke
  • Bearbeiten bestehende Getränke
  • Löschen von Getränke
FoodProductController

Die gleiche Funktionen wie der BeverageProductController.

OrderItemController

Beim Erstellen einer Bestellung wird diesen Controller verwenden, um neue OrderItems zu erstellen oder bereits erstellte zu löschen.

RestaurantOrderController

Dieser Controller ist für die Erstellung von Bestellungen zuständig, sowie vom Löschen. Diesen Controller wird bei jedem Klick auf einer offenen Bestellung oder einen freien Tisch aufgerufen.

OrderManagerController

Dieser Controller wird nach dem Login aufgerufen und erstellt eine Liste von offenen Bestellungen. Dieser Controller wird auch für die Auflistung aller Bestellungen für ein bestimmtes Datum verwendet. Defaultmässig wird den aktuellen Tag übergegeben.

UserController

Verwendet für die Verwaltung von Benützer.

Benutzerverwaltung

Für die Benutzerverwaltung wurde das Plugin SpringSecurityCore verwendet.

Die Klasse User verwaltet die Benutzer, die Zugriff auf die Applikation haben dürfen. Die Klasse Role kontrolliert die Art von Zugriff, die ein Benutzer haben darf. Aktuell sind die Role user und admin vorhande, admin kann alles machen was user kann, aber nicht umgekehrt. UserRoler ist zuständig für die Zuweisung von Roles für die Benutzer.

WICHTIG: Nur Admin darf neue Benutzer erstellen, allerdings ist dieser Controller bzw. View noch nicht implementiert.
